```python
try:
    client = get_sync_client()

    logger.error("Please verify:")
    logger.error("1. MONGO_URI is correctly set in .env file")

    logger.error("3. Network access is configured (IP whitelist)")
    logger.error("4. Database credentials are correct")

```

B. Application Startup Handling (IMPROVED)
-------------------------------------------
File: api/main.py
```python
MONGODB_AVAILABLE = False
try:
    db.init_database()
    print("[DATABASE] MongoDB initialized successfully")
    MONGODB_AVAILABLE = True
except ValueError as e:
    # Configuration error (missing environment variables)
    print(f"[DATABASE] Configuration error: {e}")
    print("[DATABASE] Application cannot start without MongoDB configuration")
    sys.exit(1)
except ConnectionError as e:
    # Connection error (can't reach MongoDB)
    print(f"[DATABASE] Connection error: {e}")
    print("[DATABASE] Application cannot start without MongoDB connection")
    sys.exit(1)
except Exception as e:
    # Unexpected error
    print(f"[DATABASE] Unexpected initialization error: {e}")
    traceback.print_exc()
    print("[DATABASE] Application cannot start without MongoDB")
    sys.exit(1)
```

Benefits:
- App fails fast with clear error message if MongoDB unavailable
- Prevents silent failures or partial functionality
- Provides troubleshooting guidance in error messages
- Distinguishes between configuration vs connection errors

9. ERROR SCENARIOS AND HANDLING
================================================================================

Scenario 1: MONGO_URI Not Set
------------------------------
Error: ValueError: MONGO_URI environment variable is not set...
Action: App exits immediately with sys.exit(1)
Solution: Set MONGO_URI in .env or environment

Scenario 2: Invalid Connection String
--------------------------------------
Error: ConnectionError: Failed to connect to MongoDB: Invalid URI...
Action: App exits immediately with sys.exit(1)
Solution: Verify MONGO_URI format and credentials

Scenario 3: Network/Firewall Issues
------------------------------------
Error: ConnectionError: Failed to connect to MongoDB: timeout...
Action: App exits after 5-10 second timeout
Solution: Check MongoDB Atlas IP whitelist, network access

Scenario 4: Wrong Database Credentials
---------------------------------------
Error: ConnectionError: Failed to connect to MongoDB: Authentication failed
Action: App exits immediately
Solution: Verify username/password in connection string

Scenario 5: MongoDB Atlas Cluster Paused/Deleted
-------------------------------------------------
Error: ConnectionError: Failed to connect to MongoDB: Server not found
Action: App exits immediately
Solution: Resume or recreate MongoDB Atlas cluster
